\section{Tips: Brevity}
\begin{itemize}
    \item Tests should be short and to the point.
    \item Long elaborate tests indicate overspecification or the code is in
        need of refactoring.
    \item Brevity helps us maintain focus on tests as documentation.
\end{itemize}
